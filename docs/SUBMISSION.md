### How It Works

#### Phase 1: Agents Provide Services
- **Wisdom Agent** 🧠 - Emotional intelligence coaching
- **Intimacy Agent** 💖 - Relationship strengthening
- **Generational Bridge** 👵👦 - Cross-age storytelling
- **Presence Agent** 🧘 - Mindfulness guidance (Presence tracking)
- **Growth Agent** 🚀 - Family challenges & resilience
- **Savings Agent** 💰 - Yield-bearing family wealth (Bonzo Finance)

#### Phase 2: Measure Impact
- Family Bond Score updated weekly from 7 signals:
  - Generational interaction frequency
  - Response reciprocity (engagement depth)
  - Sentiment trajectory (emotional tone)
  - Challenge completion (shared goals)
  - Presence consistency (active participation)
  - Network topology (relationship strength)
  - Hedera consensus (conflict resolution)

#### Phase 3: Calculate Payouts
```
Agent Payout = Base × Performance Multiplier × Recency Weight

Base = $50 × Score Delta
  (How much family improved)

Performance Multiplier = 1.0 - 1.5x
  (Rewards consistent improvement)

Recency Weight = 0.8 - 1.0
  (Weights recent impact more)

Example:
Family improves +5 points, agent has 3 consecutive wins
= $50 × 5 × 1.3 × 1.0 = $325 FAM earned
```

#### Phase 4: Prevent Gaming
**Anti-Gaming System** detects:
- Suspicious score jumps (>50 points)
- Unrealistic perfection (all metrics at 100)
- Rapid consecutive improvements (cooling period)
- Network topology anomalies

**Fair Play Enforcement:**
- Admin review of flagged payouts
- Cooling periods (1-4 weeks)
- Weekly per-agent cap ($500)
- Platform daily cap ($50,000)

#### Phase 5: Execute Payment
1. **Log to HCS** - Immutable record on Hedera Consensus Service
2. **Transfer Tokens** - FAM via Hedera Token Service
3. **Verify On-Chain** - HashScan shows all transactions
4. **Update Dashboard** - Real-time visibility

## 🚀 Deployment

### Production Architecture

**Frontend:** Netlify (SPA hosting)  
**Backend:** Hetzner VPS (artifact-based deployment)  
**API:** https://api.famile.xyz

### Quick Deploy

```bash
# One command: build + deploy
./scripts/build-artifact.sh && ./scripts/deploy-artifact.sh
```

### Deployment Strategy

**Artifact-Based with Symlinks** (no Docker overhead):
- ✅ **68% less disk** (250MB vs 800MB)
- ✅ **Atomic updates** - symlink switching
- ✅ **Instant rollback** - switch to any previous release
- ✅ **PM2** - process management with auto-restart

**Directory Structure:**
```
/opt/familexyz/
├── current -> releases/{timestamp}    # Active release
├── releases/                          # Immutable releases
└── shared/                            # Persistent state
    ├── .env                           # Environment (symlinked in)
    ├── data/                          # Database
    └── logs/                          # PM2 logs
```

### Manual Steps

**1. Configure environment on VPS:**
```bash
ssh snel-bot
nano /opt/familexyz/shared/.env
# Add: VENICE_API_KEY, HEDERA_OPERATOR_ID, HEDERA_OPERATOR_KEY
```

**2. Deploy:**
```bash
./scripts/build-artifact.sh && ./scripts/deploy-artifact.sh
```


### Rollback
